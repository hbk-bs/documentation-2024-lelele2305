Was ist ein Large Language Model?

Ein Large Language Model (LLM) ist ein großes Sprachmodell, welches auf machinellem Lernen basiert und das insbesondere auf neuronalen Netzen. Es wurde darauf trainiert, natürliche Sprache zu verstehen und zu generieren. LLMs haben Milliarden bis zu Hunderten von Milliarden Parametern. Je größer das Modell, desto komplexere Zusammenhänge kann es dann erkennen und verarbeiten.

Trainiert werden sie mit riesigen Mengen an Textdaten aus verschiedenen Quellen, wie beispielsweise Büchern, wissenschaftlichen Artikel oder Webseiten. Außerdem können sie in verschiedenen Aufgaben eingesetzt werden, da sie allgemeine Muster in der Sprache erlernen und somit auch vielseitig einsetzbar sind.

Ollama

Als Beispiel eines LLMs wird Ollama vorgestellt. Ollama ist eine Plattform, die auf lokalen Large Language Models spezialisiert ist. Sie bietet die Möglichkeit, verschiedene KI-Modelle, darunter auch LLMs, direkt auf lokalen Maschinen laufen zu lassen, ohne dass dabei eine Verbindung zu einer Cloud oder einem externen Server erforderlich ist.

Viele andere KI-Dienste setzen dagegen oft auf Cloud-basierte Lösungen. Es ermöglicht, dass Nutzer und Entwickler KI-Modelle lokal auf ihren eigenen Geräten auszuführem. Dadurch ist eine Internetverbindung für die Nutzung von LLMs umgangen. Da die Modelle lokal laufen bietet dasa außerdem einen sehr hohen Datenschutz, da keine Daten an extern Server gesendet werden. Ollama bietet eine Auswahl an leistungsstarken KI-Modellen, die in unterschiedlichen Anwendungen wie der Textgenerierung, programmatische Aufgaben oder Bilderzeugung verwendet werden können.

Wofür wird ein Large Language Model verwendet?

Das Large Language Model verwendet riesige Datenmengen von Texten, um Sprachmuster zu lernen, und kann dann Aufgaben wie das Verfassen von Texten, Übersetzen von Sprachen, Beantworten von Fragen und vieles mehr durchführen. Sie finden ihre Anwendung in Chatbots, automatisierten Übersetzungsdiensten, Textgenerierung, Programmierhilfe und vielem mehr.

Die Anwendungsbereiche:

Textgenerierung

Dazu gehören Content Kreation, Kreatives Schreiben und Ghostwriting.
Beispiele:

  • Artikel, Blogposts, Marketinginhalte, Produktbeschreibungen
  • Geschichten, Gedichte, kreatives Schreiben
  • Reden, Essays, Bücher

Kundensupport und Chatbots

Damit gemeint sind neben automatisierter Kundenberatung die rund um die Uhr Verfügbarkeit und die personalisierte Interaktion.
Beispiele:

  • Beantwortung häufig gestellter Fragen
  • Unterstützung bei technischen oder allgemeinen Anfragen durch intelligente Chatbots
  • Rund um die Uhr Kundenservice
  • Individuelle Bedürfnisse der Nutzer berücksichtigen

Übersetzung und Verarbeitung

Die Maschinelle Übersetzung, Sprachumwandlung und -analyse sind möglich.
Beispiele:

  • Übersetzung natürlicher Sprache in Andere
  • Übersetzung in bessere Qualität der Sprache
  • Text in gesprochene Sprache
  • Gesprochene Sprache in Text
  • Stimmungen erkennen und analysieren
  • Intentionen erkennen und analysieren
  • Meinung erkennen und analysieren

Programmierhilfe

LLMs ist es möglich, Entwicklern beim Schreiben des Codes zu helfen.
Beispiele:

  • Automatische Code-Vervollständigung bzw. Vorschläge
  • Fehlerbehebung und Suche nach Bugs/Problemen
  • Code-Generierung basierend auf Textbeschreibung

Bildung

Sie können nicht nur Schülern beim Lernen behilflich sein, sondern ebenfalls im Unterricht Fortschritte erzielen.
Beispiele:

  • Personalisierte Lernunterstützung für Schüler
  • Individuelle Erstellung von Übungen
  • Unterrichtsstunden
  • Zusammenfassungen schreiben

Large Language Model am Beispiel Fairytale Generator

Ein mit der Hilfe von Ollama erarbeitetes Projekt ist der Fairytale Generator.

Ist das Programm von Ollama herundergeladen, kann es mit Hilfe eines Werkzeugs zum Schreiben von Code durch das Terminal zum Laufen gebracht werden. Hier kann sich mit der gewählten Version durch den Befehl: ollama run llama3 ausgetauscht werden.

Um das LLM in den Code der geplanten Anwendung zu implementieren, besteht der erste Schritt darin, 


                        1. <script type="module">
                        2.    import ollama from "https://esm.sh/ollama/browser";
                        3.    window.ollama = ollama;
                        4.    console.log("Yes ollama is there"); 
                        5.    </script>
in den head des html-codes hinzuzufügen.

Zeile 1: Der Attributwert type="module" wird verwendet, um dem Browser mitzuteilen, dass der javaScript-Code in diesem Skript ein ECMAScript-Modul ist. Das Modul kann externe Abhängigkeiten importieren, was in diesem Fal der Ollama-Bibliothek entspricht.

Zeile 2: Der Befehl import ollama from "https://esm.sh/ollama/browser"; importiert die Ollama-Bibliothek von einem CDN namens esm.h. Dies ist ein Dienst, der es ermöglicht, NPM-Pakete im Browser zu nutzen. In diesem Fall wird die Bibliothek Ollama geladen, die für die Interaktion mit einem LLM über den Browser gedacht ist.

Zeile 3: Die Ollama-Bibiothek wird durch diese Zeile auf das globale window-Objekt gesetzt. Die Ollama-Funktion oder das Objekt werden im Browser zugägnlich gemacht, sodass andere Teile des Codes auf dieses Objekt zugreifen können. Dadurch steht ollama global zur Verfügung. Das ist nützlich, wenn die Bibliothek in verschiednen Skripten genutzt werden soll.

Zeile 4: Hier wird eine einfache Nachricht in der Browser-Konsole ausgegeben, die bestätogt, dass das Skript erfolgreich geladen wurde und die Ollama-Bibliothek vorhanden ist.

Die folgenden Bildschirmaufnahmen zeigen, mit welchen Informationen das LLM gefüttert werden kann. Im Fall des Fairytale Generators wurde es mit SVGs versorgt, die in einer bestimmten Anordnung vorkommen sollten. Ziel war es, eine Landschaft zu generieren. Das LLM sollte sich per Zufall eine der aufgeführten Landschaften aussuchen und auf Basis des zusätzlichen Textes eine kurze Geschichte schreiben, die aus 10 Sätzen bestehen sollte. Ein Klick auf den Button lässt eine neue zufällige Landschaft generieren, die ebenfalls eine neue auf den Inhalt der Landschaft basierende Geschichte schreibt.

Es folgt sehr viel Code, der die SVGs und deren Aufbau beschreibt.

Eine zufällige Landschaft wird mit Math.random() ausgewählt und dann in den landschaften-container eingefügt. Wenn window.ollama verfügbar ist, wird die Ollama API verwendet, um eine kurze Geschichte über die generierte Landschaft zu generieren. Die API-Anfrage enthält zwei Nachrichten: Eine systemweite Anweisung an das Modell und eine Benutzernachricht, die die Landschaft beschreibt. Mit der ID target wird die Antwort der API im HTML-Element angezeigt.
